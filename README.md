---

## Учебные цели

- **Базовый Java:** классы, поля, конструкторы, модификаторы доступа, `static`, блоки инициализации.
- **Конструкции языка:** `var`, абстрактные классы, интерфейсы, sealed‑типы, записи (`record`), перечисления (`enum`).
- **Стандартная библиотека:** массивы (в том числе зубчатые), строки, `StringBuilder`, коллекции (`List`, `EnumMap`, `EnumSet`).
- **Продвинутые темы:** аннотации, reflection, лямбды, ссылки на методы.

В результате практики вы должны уметь **читать** и **писать** код с этими конструкциями и **обосновывать** выбор решений.

## Требования и инструменты

- **Java:** 17+ (для `record`, sealed-интерфейсов, pattern matching в `instanceof`).
- **IDE:** IntelliJ IDEA (достаточно Community Edition).
- **Git:** вы работаете в своём репозитории, выданном через GitHub Classroom.

## Как работать с этой практикой

1. **Внимательно прочитайте** формулировки заданий Практики 2 на сайте и в `practice2.md`.  
2. В этом репозитории:
   - относитесь к коду как к **шаблону и примерам** для ваших собственных решений;
   - там, где в тексте практики нужно *проанализировать* или *сравнить* (например, модификаторы, abstract vs interface), формулируйте ответ своими словами (в тетради, отчёте, комментариях).
3. Для каждой части откройте соответствующий класс с `main` (см. таблицу ниже), прочитайте код и **запустите** его:
   - сначала попробуйте **предсказать** результат;
   - затем сравните фактический вывод с ожидаемым в задании.
4. Для теории ООП и конструкций Java откройте `oop-types-and-design-ru.md` (классы, интерфейсы, sealed, record, enum, примеры платёжной и уведомительной систем).
5. Если что‑то непонятно, откройте файл `HINTS.md`:
   - короткие напоминания теории;
   - на что обратить внимание в коде;
   - типичные ошибки, которых стоит избегать.
6. При экспериментировании с кодом **не изменяйте пакеты и сигнатуры методов**, чтобы проект оставался компилируемым и соответствовал тексту практики.

## Как запустить

Из корня проекта (папка, где лежит `src/`):

```bash
javac src/*.java src/company/core/*.java src/company/app/*.java
java -cp src Main
```

В IntelliJ IDEA: откройте эту папку, пометьте `src` как **Sources Root**, затем запускайте `Main.java` или нужный класс с `main`.

## Безопасность, аккуратность и академическая честность

- **Безопасность:** все пароли, номера счетов, IBAN и адреса в материалах **учебные**. Не храните в коде реальные секреты и ключи.
- **Аккуратность кода:** сохраняйте структуру пакетов, используйте ясные имена, форматируйте код (IntelliJ: *Code → Reformat Code*).
- **Научная и академическая корректность:** опирайтесь на официальную документацию Java и материалы курса. Термины (инкапсуляция, сокрытие информации, полиморфизм, динамическое связывание и т.д.) используйте в общепринятом значении. После изменения логики запускайте соответствующий `main` и сверяйте вывод с заданием.
- **Академическая честность:** шаблон предназначен для обучения. Пишите собственный код и формулируйте объяснения самостоятельно; при использовании внешних источников указывайте их в отчёте.
